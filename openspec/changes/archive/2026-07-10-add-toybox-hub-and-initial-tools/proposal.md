## Why

Developers accumulate small, repeated text transformations — pretty-printing JSON from a log line, pasting a runnable SQL statement out of a MyBatis log snippet — that don't justify a full extension each. ToyBox consolidates these utilities behind a single Raycast command so the catalogue can grow without polluting the user's command palette, and so that every tool follows the same clipboard-first / manual-fallback UX.

## What Changes

- Register a new `toybox` command that renders a searchable list of every tool registered in `src/tools.ts`. Selecting a tool launches its sub-command via `launchCommand`.
- Register a new `json` command that reads the clipboard on mount, pretty-prints valid JSON (2-space indent) inside a Raycast `Detail` view with a one-click copy action, and falls back to a Form with manual entry when the clipboard is empty or invalid.
- Register a new `mybatis` command that reads the clipboard, parses MyBatis `==> Preparing:` / `==> Parameters:` lines, substitutes the bound values into the SQL template, and surfaces the runnable statement in a `Detail` view with copy actions; falls back to a Form when clipboard auto-detection fails.
- Add a single source of truth (`src/tools.ts`) for the tool catalogue so adding a new utility is a one-file change.
- Add OpenSpec scaffolding (`openspec/config.yaml`, `openspec/specs/`, `openspec/changes/`) so subsequent capabilities are spec-driven.

## Capabilities

### New Capabilities
- `toybox-hub`: the hub command that lists every tool and launches the matching sub-command.
- `json-viewer`: pretty-prints JSON from the clipboard or from manual input.
- `mybatis-sql-formatter`: parses a MyBatis log snippet and returns the runnable SQL.

### Modified Capabilities
_(none — the change introduces a fresh extension.)_

## Impact

- New Raycast commands: `toybox`, `json`, `mybatis`. They appear in the Raycast command palette as separate entries so they can also be invoked directly.
- New runtime files: `src/toybox.tsx`, `src/json.tsx`, `src/mybatis.tsx`, `src/tools.ts`.
- New config entry: each command is added to the `commands` array in `package.json`.
- `raycast-env.d.ts` is updated to declare the new `Preferences.Mybatis` / `Arguments.Mybatis` namespaces.
- No new runtime dependencies; only existing `@raycast/api` and `@raycast/utils` are used.
- OpenSpec is initialised in the repo to drive future changes; existing skills (`openspec-propose`, `openspec-apply-change`, etc.) are auto-generated by `openspec init --tools codex,claude` and are gitignored.
